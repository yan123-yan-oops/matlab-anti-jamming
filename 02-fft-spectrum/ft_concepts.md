# FT / FFT 概念深度解析

> 这是你学 **02-fft-spectrum** 时最容易卡住的概念。
> 我们不说数学公式，用"感觉"来理解它。

---

## 1. FT（傅里叶变换）在干什么？

```
你听到一首歌 → 你的耳朵在干的事就是傅里叶变换
                ↓
你把"一整段声音"拆成了"什么乐器在什么时候用多大声音"
                ↓
                FT 就是把混合信号拆成一个个单一频率
```

**一句话**：FT = 把信号从"随时间变化"翻译成"随频率变化"。

| 域 | 横轴 | 看的什么 |
|----|------|----------|
| 时域 | 时间 (s) | 信号怎么随时间变 |
| 频域 | 频率 (Hz) | 信号里有哪些频率成分 |

---

## 2. FFT（快速傅里叶变换）不是另一种变换

**FFT 就是 FT 的"快速算法"**，就像乘法分配律比一个个加要快一样：

```
FT 的计算量：N² 次运算
FFT 的计算量：N·log₂(N) 次运算

N = 1024 时：
  FT  ≈ 1,000,000 次
  FFT ≈ 10,000 次  ← 快 100 倍！
```

**FFT 要求输入长度是 2 的幂**（64、128、256、512、1024...），
和 C 里环形缓冲区为什么要是 2 的幂是**同一个原因**——可以用位运算代替取模。

---

## 3. 采样率、点数、频率分辨率三者的关系

```
频率分辨率 = 采样率 / FFT点数

例：Fs = 1000Hz, N = 1000 点
  分辨率 = 1000/1000 = 1 Hz → 能区分 10Hz 和 11Hz
  
例：Fs = 1000Hz, N = 100 点
  分辨率 = 1000/100 = 10 Hz → 只能区分 10Hz 和 20Hz
```

**怎么提高分辨率？**
- 增加采样时长（采更多点）→ FFT 点数增大 → 分辨率提高 ✨

---

## 4. 频谱泄露 —— 为什么谱线会"糊掉"

```
你切了一小段信号来做 FFT
                ↓
             这相当于"突然开始、突然结束"
                ↓
            信号被"截断"了 → 能量扩散到旁边频率
                ↓
              这就是频谱泄露
```

**加窗 = 让开始和结束不那么突然**（像音量渐入渐出）

| 窗函数 | 主瓣宽度 | 旁瓣高度 | 适合场景 |
|--------|----------|----------|----------|
| 矩形（不加窗） | 最窄 | 最高 | 要求分辨率 |
| 汉宁 (hann) | 中等 | 低 | 通用推荐 |
| 布莱克曼 | 最宽 | 最低 | 要求抑制噪声 |

## 5. 和 C 指针的联系（对应你的学习路线）

| C 指针概念 | MATLAB FFT 概念 | 共同点 |
|-----------|----------------|--------|
| `arr[i]` = 取第 i 个元素 | `X(k)` = 第 k 个频率分量 | 都是索引偏移 |
| `arr + 1` 跳过一个元素大小 | `k+1` 跳过一个频率分辨率 | 间隔由元素大小/分辨率决定 |
| 数组长度 = 分配的内存大小 | FFT 点数 = 频率分辨率 | 决定了你能"看到"多少东西 |

---

## 6. 常见困惑速查

| 困惑 | 答案 |
|------|------|
| FFT 结果为什么对称？ | 实信号的频谱是共轭对称的，只看前一半即可 |
| 横轴是 0~Fs？还是 -Fs/2~Fs/2？ | 都行，`fftshift` 把 0 频移到中间 |
| 幅度怎么算？ | `abs(X)/N*2`（单边谱，除直流外都 ×2） |
| 相位怎么看？ | `angle(X)`，但实际工程很少看相位 |
| 补零能提高分辨率吗？ | ❌ 不能！补零只是插值，真正的分辨率由采样时长决定 |
